Pulumify

Pratik Deoghare

When you use Pulumi with TypeScript, you eventually run into a mismatch: external libraries return plain types, while Pulumi works with Input and Output to track dependencies.

Consider a simple example. An external library defines a type: 

type Task = {
    Type: 'Task'
    Resource: string
}

Now you want to assign a Lambda ARN generated by Pulumi: 

const lambdaArn = new random.RandomString("mine", {
    length: 10,
});

const a : Task = {
    Type: 'Task',
    Resource: lambdaArn.id, // ERROR here.
}

This fails: 

error TS2322: Type 'Output<string>' is not assignable to type 'string'.

You could use .apply, but there is a more elegant way.

We can define a type that makes every field accept either a raw value or a Pulumi input: 

type Pulumify<T> = {
    [K in keyof T]: T[K] extends object
        ? Pulumify<T[K]> | pulumi.Input<Pulumify<T[K]>>
        : T[K] | pulumi.Input<T[K]>;
};

Now we can write: 

const a: Pulumify<Task>  = {
    Type: 'Task',
    Resource: lambdaArn.id,
};
No error.

This works because Pulumify<T> takes a type and recursively constructs a new type where every field with type T now accepts T | pulumi.Input<T> . 

type Task = {
	Type: 'Task'
	Resource: string
}

after pulumification becomes 

type Task = {
	Type: 'Task' | pulumi.Input<'Task'>
	Resource: string | pulumi.Input<string>
}
